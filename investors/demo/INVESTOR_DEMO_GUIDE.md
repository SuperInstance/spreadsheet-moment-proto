## Demo Best Practices

### Preparation
- **Practice:** Rehearse demo 10+ times until natural
- **Backup:** Have offline backup (screenshots, video)
- **Data:** Use realistic demo data (not Lorem Ipsum)
- **Environment:** Use staging environment (not production)
- **Timing:** Keep demo under 20 minutes total

### During Demo
- **Engage:** Ask questions, check for understanding
- **Pacing:** Don't rush, pause for emphasis
- **Stories:** Use stories to make points memorable
- **Interactive:** Let investor drive if interested
- **Honest:** Admit if something doesn't work

### After Demo
- **Follow-up:** Send demo link immediately after
- **Feedback:** Ask for feedback and questions
- **Next Steps:** Propose specific next steps
- **Materials:** Attach deck and additional info

## Demo Troubleshooting

### Common Issues

**Issue:** Demo environment is slow
- **Solution:** Switch to pre-recorded video, apologize, move on

**Issue:** Internet connection fails
- **Solution:** Switch to offline backup (screenshots), use mobile hotspot

**Issue:** Agent makes a mistake
- **Solution:** Explain confidence scoring, show fallback to LLM, emphasize hybrid approach

**Issue:** Investor asks about feature not built yet
- **Solution:** Acknowledge, show roadmap, explain prioritization framework

**Issue:** Technical question you can't answer
- **Solution:** "Great question, let me follow up with our CTO and get back to you"
