## The Pollination Metaphor

### Why Bees?

| Bee Property | POLLN Parallel |
|--------------|----------------|
| Individual bees are simple | Individual agents are narrow |
| Colony is intelligent | Colony produces intelligent behavior |
| Pollination is passive | Learning happens as side effect |
| Honey is valuable output | Cells (routines) are valuable output |
| Hives persist across bee generations | Knowledge persists across sessions |
| Waggle dance shares knowledge | Scent trails share patterns |
| Random pollination finds new combinations | Random exploration finds novel solutions |

### Why Not Fungi?

| Issue | Mycelium Metaphor | Pollination Metaphor |
|-------|-------------------|---------------------|
| Agency | Fungi are passive | Bees have clear purpose |
| Sharing | Resource sharing only | Intentional knowledge sharing |
| Evolution | Slow, genetic | Fast, memetic |
| Output | Mushrooms appear randomly | Honey is actively produced |
| Human relationship | Forest floor (distant) | Hive (tended, known) |

## Vocabulary Design Principles

### 1. Evocative Precision
Names should:
- Evoke the right image
- Be precise enough for technical use
- Be memorable
- Connect to the metaphor

### 2. Layered Meaning
Best names work on multiple levels:
- **Surface**: What it does
- **Metaphor**: How it connects to bees
- **Technical**: Implementation hint

Example: **Pollen Grain**
- Surface: A small packet of information
- Metaphor: Portable, potent, can cross-pollinate
- Technical: Compressed embedding (64-1024 floats)

### 3. Consistent Register
All terms should feel like they belong to the same conceptual world:
- Colony, not Forest
- Keeper, not Gardener
- Meadow, not Grove
- Cell, not Loom

### 4. Cultural Sensitivity
- Avoid appropriating sacred terms
- Credit origins where due
- Respect indigenous knowledge systems

## Naming Workshop Process

### Phase 1: Brainstorm
Generate many candidates without judgment

### Phase 2: Filter
Apply criteria:
- [ ] Evocative?
- [ ] Precise?
- [ ] Memorable?
- [ ] Metaphor-consistent?
- [ ] Culturally appropriate?

### Phase 3: Test
- Say it aloud
- Use it in a sentence
- Explain it to someone new
- Check for unintended meanings

### Phase 4: Document
- Definition
- Etymology/origin
- Usage examples
- Related terms

## Style Guide

### Voice
- Warm but technical
- Organic but precise
- Accessible but not dumbed down

### Examples

**Good**:
> "Your colony learns from every action. Like bees returning to the hive with nectar, each trace enriches your colony mind."

**Bad**:
> "The AI model processes user input to update the knowledge base." (Too clinical)

**Also Bad**:
> "Your little bee friends buzz around collecting sweet wisdom nectar!" (Too cute)

### Tense
- Present for what the system does: "Your colony learns"
- Future for what will happen: "Overnight, your colony will process"
- Never passive: Not "Traces are logged" but "Your colony logs traces"

### Person
- Second person for keeper: "Your colony"
- Third person for system: "The colony network"
- First person collective: "We believe" (for documentation)

## Cultural Sensitivity Checklist

Before finalizing any term:

- [ ] Does it reference a specific culture? If so, is attribution given?
- [ ] Is it sacred or restricted in any tradition?
- [ ] Could it be seen as appropriation?
- [ ] Does it respect the FPIC protocol?
- [ ] Has the Indigenous Liaison reviewed it?

### Terms That Require Attribution

| Term | Origin | Attribution |
|------|--------|-------------|
| Garden metaphor | Pan-indigenous | "Inspired by indigenous cultivation practices" |
| Mycelial network concept | TEK | "Known to Traditional Ecological Knowledge systems" |
| Relational ontology | Multiple | "Present in many non-Western philosophies" |

### Terms to Avoid

| Avoid | Why | Alternative |
|-------|-----|-------------|
| Shaman | Sacred, often misused | Practitioner |
| Spirit animal | Sacred, often trivialized | Companion pattern |
| Totem | Sacred | Symbol |
