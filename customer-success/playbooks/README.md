## Overview

This playbook collection provides scenario-based guides for Customer Success Managers (CSMs) to handle common customer situations, drive adoption, prevent churn, and identify expansion opportunities. Each playbook includes triggers, goals, action steps, scripts, and success criteria.

## Trial Conversion Playbook

**Scenario:** Free trial user approaching trial end date

**Trigger:**
- Trial user at day 21 (7 days before expiration)
- Trial user with high engagement
- Trial user requesting enterprise features

**Objectives:**
- Convert trial to paid subscription
- Ensure value realization before trial end
- Establish long-term relationship

**Success Criteria:**
- Trial to paid conversion rate > 25%
- Time to conversion < 45 days
- Post-conversion retention > 90%

### Action Steps

**Step 1: Assessment (Day 21)**
```
1. Review trial user engagement
   - Login frequency
   - Feature usage
   - Spreadsheet creation
   - Collaboration activity

2. Evaluate value realization
   - Time to first value achieved?
   - Key features adopted?
   - Integration completed?
   - Team onboarded?

3. Identify conversion barriers
   - Technical issues
   - Budget concerns
   - Feature gaps
   - Timing issues
```

**Step 2: Personalized Outreach (Day 21-22)**
```
1. Send conversion-focused email
   - Highlight achievements
   - Emphasize value received
   - Address common concerns
   - Provide clear next steps

2. Offer personalized consultation
   - Review usage and successes
   - Answer questions
   - Address concerns
   - Recommend optimal plan

3. Provide conversion incentive
   - Discount code (if needed)
   - Extended trial (if justified)
   - Bonus features
   - Priority onboarding
```

**Step 3: Value Reinforcement (Day 23-25)**
```
1. Share success metrics
   - Spreadsheets created
   - Time saved
   - Collaborators invited
   - Features used

2. Provide advanced feature preview
   - Upcoming features
   - Beta program access
   - Roadmap preview
   - Feature voting

3. Facilitate decision
   - Plan comparison
   - ROI calculation
   - Payment options
   - Contract terms
```

**Step 4: Conversion Support (Day 26-28)**
```
1. Streamline conversion process
   - One-click upgrade
   - Invoice generation
   - Team add-on process
   - SSO setup (enterprise)

2. Confirm value continuation
   - No data loss
   - Feature continuity
   - Support access
   - Team preservation

3. Celebrate conversion
   - Welcome email
   - On-success check
   - Success manager assignment
   - Resource provision
```

## User Activation Playbook

**Scenario:** New user needs rapid value realization

**Trigger:**
- User signs up (immediate)
- User creates first spreadsheet
- User shows early engagement signals

**Objectives:**
- Achieve time-to-first-value < 15 minutes
- Drive feature adoption to 3+ features in week 1
- Establish daily usage habit

**Success Criteria:**
- Activation rate > 75%
- Week 1 retention > 80%
- Feature adoption > 3 features

### Action Steps

**Step 1: Immediate Welcome (T+0)**
```
1. Send welcome email (automated)
2. Trigger in-app tour
3. Suggest first spreadsheet template
4. Offer assistance channel
```

**Step 2: First Value Achievement (T+0-24h)**
```
1. Monitor first spreadsheet creation
2. Track feature exploration
3. Identify potential roadblocks
4. Send contextual tips
```

**Step 3: Habit Formation (Day 2-7)**
```
1. Daily tip emails (Day 2, 4, 6)
2. Feature deep-dive suggestions
3. Success celebration emails
4. Community invitation
```

**Step 4: Adoption Expansion (Day 8-14)**
```
1. Advanced feature introduction
2. Template library exploration
3. Collaboration encouragement
4. Power user pathway identification
```

### Activation Checklist

**Day 1 Checklist:**
- [ ] Account created and verified
- [ ] First spreadsheet created
- [ ] Basic data entered
- [ ] First formula used
- [ ] Help center visited
- [ ] Onboarding tour completed

**Week 1 Checklist:**
- [ ] 3+ spreadsheets created
- [ ] AI features used
- [ ] Template explored
- [ ] Help article accessed
- [ ] Second session completed
- [ ] Community joined

### Monitoring Points

**Early Warning Signs:**
- No first spreadsheet in 2 hours
- Tour abandonment
- Error messages encountered
- Support ticket opened

**Intervention Triggers:**
- No activity for 24 hours
- Incomplete first spreadsheet
- Repeated help searches
- Negative feedback

## Renewal Playbook

**Scenario:** Subscription approaching renewal date

**Trigger:**
- 90 days before renewal (enterprise)
- 30 days before renewal (team)
- 14 days before renewal (individual)

**Objectives:**
- Secure renewal at current or higher tier
- Identify expansion opportunities
- Gather feedback and testimonials

**Success Criteria:**
- Renewal rate > 90%
- Expansion revenue > 10% of renewing accounts
- NPS contribution from renewing accounts

### Action Steps

**Step 1: Renewal Preparation (90 days out)**
```
1. Account health assessment
   - Review engagement metrics
   - Check support history
   - Analyze usage patterns
   - Identify champions

2. Value quantification
   - Compile usage statistics
   - Calculate time savings
   - Document success stories
   - Identify key wins

3. Renewal strategy planning
   - Determine approach
   - Prepare talking points
   - Identify expansion opportunities
   - Address potential objections
```

**Step 2: Value Review (60 days out)**
```
1. Schedule renewal discussion
   - Propose meeting agenda
   - Include key stakeholders
   - Prepare value presentation
   - Gather testimonials

2. Conduct value review meeting
   - Present achievements
   - Review ROI data
   - Discuss future needs
   - Identify expansion areas

3. Document feedback and requests
   - Feature requests
   - Process improvements
   - Concerns addressed
   - Success stories
```

**Step 3: Renewal Proposal (30 days out)**
```
1. Prepare renewal proposal
   - Current plan review
   - Value delivered summary
   - Expansion recommendations
   - Renewal terms and pricing

2. Send renewal proposal
   - Value statement
   - Renewal options
   - Expansion opportunities
   - Action timeline

3. Follow-up and negotiation
   - Address questions
   - Handle objections
   - Customize terms if needed
   - Finalize agreement
```

**Step 4: Renewal Confirmation (14-7 days out)**
```
1. Secure renewal commitment
   - Contract signing
   - Payment method update
   - Team seat confirmation
   - Start date confirmation

2. Renewal celebration
   - Thank customer
   - Acknowledge partnership
   - Share upcoming features
   - Invite to customer advisory
```

### Meeting Agenda Template

**QBR/Renewal Meeting Agenda (60 minutes):**

**Opening (5 min)**
- Welcome and objectives
- Agenda review
- Participant introductions

**Value Review (20 min)**
- Usage statistics overview
- Key achievements highlight
- ROI calculation presentation
- Success story sharing

**Future Planning (20 min)**
- Upcoming needs discussion
- Feature roadmap alignment
- Expansion opportunity identification
- Goal setting for next period

**Renewal Discussion (10 min)**
- Renewal terms review
- Expansion proposal
- Questions and concerns
- Next steps confirmation

**Closing (5 min)**
- Summary and commitment
- Timeline confirmation
- Thank you and appreciation

## Upsell Playbook

**Scenario:** Customer ready for expansion

**Trigger:**
- Upsell readiness score > 80
- Plan limit approaching (90%+)
- Feature request for premium capability
- Organizational expansion signals

**Objectives:**
- Identify expansion needs
- Present relevant solutions
- Close upsell opportunity
- Increase customer value

**Success Criteria:**
- Upsell conversion rate > 40%
- Expansion revenue growth > 15% QoQ
- Post-upsell retention > 95%

### Upsell Qualification Framework

**BANT Framework:**

**Budget**
- "What's your budget for this expansion?"
- "When does your fiscal year start?"
- "Who needs to approve this?"

**Authority**
- "Who else will be involved in this decision?"
- "What's your procurement process?"
- "Do you have budget autonomy?"

**Need**
- "What problem will this solve?"
- "What's the impact of not solving it?"
- "What's driving this need now?"

**Timeline**
- "When do you need this in place?"
- "What's driving this timeline?"
- "What happens if this delays?"

### Upsell Conversation Guide

**Opening:**
```
"Based on our conversation about [current need], and seeing how you've been
using [current feature], I think you're ready for [expanded capability]. This
would allow you to [specific benefit]."
```

**Value Presentation:**
```
"Teams in your situation typically see:
- 40% increase in productivity
- 60% reduction in manual work
- 3x faster project completion

Based on your current usage, I estimate you'd see [specific benefit]."
```

**Handling Objections:**

**Budget:**
```
"I understand budget is a concern. Let's look at the ROI:
- Current cost: $X
- Time savings: X hours
- Value of time: $Y
- Net benefit: $Z

The expansion pays for itself in [timeframe]. Would you like me to prepare a
detailed ROI analysis?"
```

**Timing:**
```
"I appreciate timing isn't ideal. However, delaying until [next period] means
missing out on [immediate benefit]. We can structure this to [payment option]
if that helps. What would make this workable now?"
```

**Need:**
```
"You're right to question whether this is necessary. Let me show you the data
on similar teams who adopted this capability. [Case study]. Based on your
current usage patterns, you're likely hitting limitations already. How much
is this currently impacting your team?"
```

## At-Risk Intervention Playbook

**Scenario:** Customer showing churn risk signals

**Trigger:**
- Health score drops below 60
- Usage decline > 50% (7 days)
- No activity 14+ days
- Negative survey response
- Competitor mention

**Objectives:**
- Identify root cause of disengagement
- Address concerns and remove barriers
- Re-engage customer with value
- Prevent churn

**Success Criteria:**
- At-risk recovery rate > 60%
- Health score improvement > 20 points
- Activity resumption within 14 days

### Intervention Process

**Step 1: Risk Assessment**
```
1. Analyze risk indicators
   - Health score trend
   - Usage decline pattern
   - Support ticket history
   - Sentiment analysis

2. Identify root cause
   - Technical issues
   - Value gap
   - Competitive pressure
   - Organizational change

3. Determine intervention strategy
   - Technical support needed
   - Value reinforcement required
   - Feature gaps to address
   - Relationship rebuilding
```

**Step 2: Outreach Strategy**
```
1. Choose appropriate channel
   - High risk: Phone call
   - Medium risk: Personalized email
   - Low risk: Automated check-in

2. Craft message
   - Acknowledge concern
   - Express commitment
   - Request feedback
   - Offer assistance

3. Set follow-up
   - Schedule call/meeting
   - Provide resources
   - Create action plan
   - Establish check-in cadence
```

**Step 3: Value Reinforcement**
```
1. Re-educate on value
   - Forgotten features
   - Underutilized capabilities
   - New features since adoption
   - Success stories

2. Remove barriers
   - Technical issues resolved
   - Feature requests documented
   - Training provided
   - Configuration optimized

3. Re-engage with wins
   - Quick wins identified
   - Success celebrated
   - Progress tracked
   - Momentum rebuilt
```

**Step 4: Ongoing Monitoring**
```
1. Close monitoring period (30 days)
   - Daily health score checks
   - Weekly activity reviews
   - Bi-weekly check-ins
   - Rapid response to issues

2. Adjust as needed
   - Additional support
   - Feature accommodation
   - Plan adjustment if warranted
   - Executive sponsorship if needed
```

## Customer Advocacy Playbook

**Scenario:** Identify and cultivate customer advocates

**Trigger:**
- NPS score 9-10
- High product engagement
- Community participation
- Positive feedback shared

**Objectives:**
- Transform satisfied customers into advocates
- Leverage advocates for growth
- Build advocate community
- Reward advocacy behavior

**Success Criteria:**
- Advocate identification rate > 20% of promoters
- Advocate participation rate > 60%
- Referral rate from advocates > 3:1
- Case study contribution > 10% of advocates

### Advocacy Development Framework

**Stage 1: Identification**
```
Identify potential advocates through:
- NPS surveys (score 9-10)
- Product engagement (health score 80+)
- Community activity (forums, events)
- Positive support interactions
- Social media mentions
- Feature constructiveness
```

**Stage 2: Cultivation**
```
Nurture potential advocates:
- Personal thank you messages
- Exclusive feature previews
- Behind-the-scenes access
- Input on product roadmap
- Direct product team access
- Recognition in community
```

**Stage 3: Activation**
```
Engage advocates in activities:
- Case study participation
- Reference calls
- Event speaking
- User group leadership
- Content creation
- Beta testing
- Community moderation
```

**Stage 4: Recognition**
```
Reward and recognize advocates:
- Advocate badge/certification
- Exclusive events
- Feature credits
- License upgrades
- Public recognition
- Professional networking
- Career opportunities
```

### Advocacy Opportunities

**Light Engagement (Low commitment):**
- Product reviews (G2, Capterra)
- Social media mentions
- Community forum participation
- Feedback survey completion
- NPS survey participation

**Medium Engagement (Medium commitment):**
- Case study interviews
- Reference calls (occasional)
- Webinar guest appearance
- Blog post contribution
- User group attendance
- Beta program participation

**Deep Engagement (High commitment):**
- Case study video production
- Conference speaking
- Reference calls (frequent)
- User group leadership
- Community moderation
- Advisory board membership
- Press interview participation
