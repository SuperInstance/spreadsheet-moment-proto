# Test Script: Pythagorean Snap Calculator
**Moderator Guide - Round 12**

## Session Setup
- Open the Pythagorean Snap Calculator link: [superinstance.ai/demos/interactive#pythagorean-snap]
- Ensure screen sharing is active
- Start recording
- Read appropriate intro based on participant's experience

---

## Introduction Script (2 minutes)

"This is a tool for exploring right triangles, specifically those that follow the Pythagorean theorem - that's A² + B² = C². We'll be looking at 'Pythagorean triples' which are sets of whole numbers that make perfect right triangles, like 3-4-5. Think of them as the 'easy snaps' for geometry. As always, please think aloud as you explore."

---

## Task 1: Initial Discovery (4 minutes)
**Objective:** Assess math comfort and prior knowledge

### Instructions
"Please spend a few minutes exploring this calculator. Show me what you discover and tell me what you think this tool is for."

### What to Observe
✅ Comfort level with mathematical concepts
✅ Whether they recognize triangle visualization
✅ Understanding of the properties shown
✅ Natural engagement with controls

### Key Questions to Follow Up
- "What do you notice about the triangle shown?"
- "What mathematical relationship is this demonstrating?"
- "How might this tool be useful?"

### Note Taking
- Math background confidence level: High/Medium/Low
- Prior knowledge of Pythagorean theorem: Yes/No/Partial
- Natural curiosity: High/Medium/Low

---

## Task 2: Preset Exploration (4 minutes)
**Objective:** Test preset functionality comprehension

### Instructions
"Try selecting different Pythagorean triples from the dropdown menu. Tell me what you notice as you change between them."

### What to Observe
✅ Can they locate and use the dropdown menu
✅ Do they notice the visual changes to the triangle
✅ Do they observe how the properties change
✅ Whether scaling makes sense to them

### Success Criteria
- Successfully switches between at least 3 presets
- Notices changes in triangle size or shape
- Understands relationship between sides
- Shows interest in different use cases

### Follow-up Prompts
- "Which triple do you think would be most practical for real-world use?"
- "Any surprises in the property values?"
- "What patterns do you notice in the numbers?"

---

## Task 3: Custom Values Challenge (5 minutes)
**Objective:** Test the "snap to nearest Pythagorean" feature

### Part A: Enter Custom Values
"Now click the box to use custom values. Enter 7 for side A, and leave side B blank. What happens?"

### What to Observe
✅ Can they find the custom input option
✅ Whether they understand the interface changes
✅ How they interpret the results

### Part B: Snap Feature
"Now click 'Snap to Nearest Pythagorean.' What do you think just happened?"

### What to Observe
✅ Notices the calculated values change
✅ Understands the side B was calculated
✅ Recognizes this helps make perfect triangles
✅ Can generalize the concept

### If Confused, Ask
- "What numbers replaced the blank?"
- "Why might those particular numbers have been chosen?"
- "How does the new triangle compare to the old one?"

---

## Task 4: Scale Manipulation (3 minutes)
**Objecive:** Understand scaling concept

### Instructions
"Try using the scale feature. Start with different scales and see what changes."

### What to Observe
✅ Locates scale control
✅ Understands scaling increases/decreases size
✅ Recognizes area changes quadratically
✅ Connects scaled versions to base sizes

### Success Indicators
- Tests multiple scale values
- Identifies how scaling affects each property
- Shows understanding that ratios stay constant

### Common Difficulties to Note
- Confusing linear vs area scaling
- Not noticing 3D applications (construction)
- Missing the practical applications

---

## Task 5: Visualization Analysis (4 minutes)
**Objective:** Deep understanding of visual representation

### Instructions
"Look at the SVG visualization carefully. Tell me everything this diagram is communicating."

### What to Observe
✅ Can identify right angle indicator
✅ Recognizes side length labels
✅ Understands angle measurements
✅ Sees the animated elements
✅ Recognizes practical construction application (G-code)

### Guiding Questions
- "What does the little square in the corner represent?"
- "How are the angles shown?"
- "What might those dotted lines indicate?"
- "Who would find this visualization useful?"

### Probing Deeper
- "If you were cutting wood for this triangle, what measurements would you need?"
- "How would you verify this was actually a right triangle in real life?"

---

## Task 6: Practical Application (3 minutes)
**Objective:** Transfer learning to real-world scenario

### Instructions
"Imagine you need to build a ramp that goes up 4 feet while extending 10 feet horizontally, and it must meet safety codes requiring strong support. How could you use this tool?"

### Key Points to Grasp
- Need to find hypotenuse for support beam
- Safety codes often specify minimum ratios
- Actual measurements might need scaling
- Could check alternatives nearby

### Success Signs
- Recognizes need to find hypotenuse
- Understands how to input custom values
- May suggest checking multiple options
- Shows engineering thinking about support

---

## Final Assessment (2 minutes)

### Rating Questions
1. "On a scale of 1-5, how useful do you think this tool is for learning geometry?"
2. "What was the most interesting thing you discovered?"
3. "What was confusing or unclear?"
4. "Who should use this tool - students, teachers, builders, or all three?"
5. "Any suggestions for making it easier to understand?"

### Open Ended
"If you were going to teach someone about right triangles using only this tool, what would be the first thing you'd show them?"

---

## Scoring Rubric

### Mathematical Understanding
- **Excellent (5):** Explains Pythagorean theorem, recognizes all properties, sees scaling applications
- **Good (4):** Understands right triangles, knows terminology, gets basic concepts
- **Fair (3):** Follows along but needs prompting, understands visual but not underlying math
- **Poor (2):** Struggles with concepts, doesn't see triangle relationships
- **Very Poor (1):** Cannot explain any mathematical aspects

### Tool Usability
- **Excellent (5):** Navigates fluidly, discovers features independently, intuitive use
- **Good (4):** Successful with minor assistance, explores naturally
- **Fair (3):** Needs guidance but eventually successful, some confusion
- **Poor (2):** Multiple assists required, misses key features
- **Very Poor (1):** Cannot use tool effectively even with help

### Engagement/Interest
- **Excellent (5):** Asks questions, explores beyond tasks, shows genuine interest
- **Good (4):** Completes tasks willingly, some spontaneous exploration
- **Fair (3):** Does required tasks with some interest
- **Poor (2):** Minimal effort, wants to finish quickly
- **Very Poor (1):** Disengaged, confused, or frustrated

---

## Special Notes

### For Low Math Users
- Emphasize visual learning
- Don't use technical jargon
- Focus on "why this is useful"
- Connect to everyday examples they know

### For Math Teachers
- Ask about classroom applications
- Evaluate pedagogical value
- Request teaching suggestions
- Consider age-appropriate adaptations

### For Engineers/Builders
- Focus on practical applications
- Discuss precision requirements
- Explore G-code/cutting implications
- Connect to professional workflows

**End of Script** - Remember participant comfort over data collection!
