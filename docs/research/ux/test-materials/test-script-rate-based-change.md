# Test Script: Rate-Based Change Simulator
**Moderator Guide - Round 12**

## Session Setup
- Open the Rate-Based Change Simulator link: [superinstance.ai/demos/interactive#rate-based-change]
- Ensure screen sharing is active
- Start recording
- Read appropriate intro based on math background

---

## Introduction Script (2 minutes)

"This tool helps visualize how values change over time when they're controlled by different rates of change. You might use it to model population growth, investment returns, or really any system where something changes at a certain speed. The core formula is shown at the top: x(t) = x₀ + ∫r(τ)dτ, which means the value at time t equals the starting value plus the accumulation of all the rates up to that time. Don't worry if the formula looks complex - we'll explore it visually."

---

## Pre-Assessment Note
**Check participant comfort with:**
- Basic algebra (yes/no)
- Charts/graphs (yes/no)
- Calculus concepts (yes/no)

This determines how deeply to discuss the math.

---

## Task 1: Initial Scan (3 minutes)
**Objective:** Understand first impressions and identify key interface elements

### Instructions
"Please take a look at this tool and tell me what you see. Describe the different parts of the interface and what you think they're for."

### What to Observe
✅ Notices the chart/graph visualization
✅ Identifies the rate function selection
✅ Recognizes the parameter inputs
✅ Understands the 3-column layout concept
✅ Weather they seem overwhelmed or intrigued

### If They Miss Key Elements
- "What do you think the gray area represents?"
- "Do you see any prediction elements?"
- "What controls can you identify?"

### Note Taking Cues
- Confidence level: High/Medium/Low
- Mathematical background evident: Yes/Partial/No
- Initial chart understanding: Intuitive/Confusing

---

## Task 2: Static Rate Exploration (4 minutes)
**Objective:** Understand rate function selection and static data interpretation

### Instructions
"Start with the Constant Rate function. Tell me what happens in the visualization and what story the chart is telling."

### What to Observe
✅ Can select the right function
✅ Interprets flat data correctly
✅ Understands value increases linearly
✅ Recognizes no rate variation
✅ Can read both the rate and value data

### Success Indicators
- Picks "Constant Rate" option
- Explains value increases steadily
- Notes rate stays the same
- May comment on real-world examples (like fixed interest)

### Probing Questions
- "What's happening to the rate over time?"
- "How is this different from a value that would appear as a flat line?"
- "Where might you see this kind of change in real life?"

### During Visualization
"Now we'll simulate this dynamically. Watch what happens as time progresses."

**Observe:**
- Reaction to animation
- Understanding of rate vs value difference
- Comments on speed/pacing

---

## Task 3: More Complex Rates (5 minutes)
**Objective:** Compare different rate functions and understand their effects

### Part A: Linear Growth
"Switch to Linear Growth and watch the simulation. Describe what you see changing."

### Part B: Your Choice
"Now pick any other rate function you find interesting. Run it and tell me what makes it different from what we just saw."

### What to Observe
✅ Successfully switches between functions
✅ Understands rate is changing, not constant
✅ Recognizes acceleration concepts (quadratic/exponential)
✅ Understands periodic behavior (sinusoidal)
✅ Can generally predict increasing vs decreasing trends

### Key Understanding Checkpoints
- **Linear:** "Slope gets steeper" recognition
- **Exponential:** "Growing faster and faster"
- **Oscillating:** "Up and down" pattern
- **Square wave:** Discrete jumps between values

### Real-World Connection Prompts
- "What might cause exponential growth in business?"
- "Where would you see wave-like changes?"
- "What system behaves with sudden jumps?"

---

## Task 4: Parameter Manipulation (5 minutes)
**Objective:** Test interactive parameter understanding

### Instructions
"The initial value shows how much we start with. Change it from 100 to 50 and tell me what difference you observe."

### Extended Task
"Now experiment with the rate constant. How does changing it affect the simulation?"

### What to Observe
✅ Locates parameter controls
✅ Understands initial value shifts curve vertically
✅ Recognizes rate constant affects speed/acceleration
✅ Can predict general effect before running simulation
✅ No confusion between parameters

### Expected Understanding Progression
1. Initial value = starting point
2. Rate constant = speed/power of change
3. Integration: Starting + accumulation = current

### Success Signs
- Correctly explains vertical shift
- Connects rate constant to "how fast" change happens
- Can give rough predictions
- Understands parameters work together

### For Advanced Users Only
If they understand quickly: "What do you think happens mathematically when these parameters combine?"

---

## Task 5: Error Bounds and Prediction (4 minutes)
**Objective:** Assess understanding of uncertainty modeling

### Part A
"We'll continue running this simulation. Watch as time goes by and see if you notice anything about prediction accuracy."

### Part B
"There's a checkbox for 'Show Error Bounds'. Please turn that on and explain what this new visualization means."

### What to Observe
✅ Notices predictions don't match exactly
✅ Can interpret gray area as uncertainty/confidence range
✅ Understands why uncertainty grows over time3
✅ Grasp of real-world prediction limitations

### Key Concepts to Check
- Past is certain, future is estimated
- Longer prediction = more uncertainty
- Model acknowledges imperfection
- Different from making a "wrong" prediction

### Probing Questions
- "Why might the actual path differ from predictions?"
- "What would make predictions more accurate?"
- "When would you care about this uncertainty?"

---

## Task 6: Export and Analysis (3 minutes)
**Objective:** Check awareness of data export functionality

### Instructions
"There's an Export Data button. When and why might someone want to save this data?"

### What to Observe
✅ Finds the export button
✅ Understands data could be analyzed further
✅ Connects to spreadsheet or statistical analysis tools
✅ Recognizes professional applications

### Follow-up Prompts
- "What would you do with this exported file?"
- "Who would find having this data useful?"
- "How might you share this analysis with others?"

---

## Task 7: Real-World Problem Solving (4 minutes)
**Objective:** Test transfer learning to practical scenarios

### Instructions
"Imagine you manage a small business. Sales have been growing linearly for the past year, starting at $5,000/month and increasing by about $200/month. Use this tool to show me what your sales might look like in 18 months, and how confident you could be in that prediction."

### What to Observe
✅ Can set appropriate initial value
✅ Chooses correct rate function (linear)
✅ Adjusts rate constant correctly
✅ Interprets error bounds in business context
✅ Offers practical insights

### Support if Needed
- "What would be your initial monthly sales?"
- "How much would you estimate increasing each month?"
- "How far into the future should we look?"

### Success Criteria
- Sets realistic parameters
- Runs simulation appropriately
- Provides business insight
- Shows understanding of uncertainty in revenue planning

---

## Final Assessment (3 minutes)

### Rating Questions
1. "Rate your understanding of rate-based change from 1-5"
2. "What was the most intuitive part of this tool?"
3. "What was most confusing?"
4. "How could we better explain the mathematical concepts?"
5. "Who would benefit most from using this?"

### Application Questions
"If you had to explain what you learned to a friend, what would you tell them?"

"What would you add or change to make this more useful for students or professionals?"

---

## Scoring Guide

### Mathematical Concept Comprehension
- **Excellent (5):** Understands rate VS value, integration concept, uncertainty modeling
- **Good (4):** Gets rate functions and predictions, minor gaps in integration understanding
- **Fair (3):** Understands visual patterns but struggles with underlying math
- **Poor (2):** Sees graphs as decorations, cannot explain rate concepts
- **Very Poor (1):** Completely confused by visualizations and controls

### Tool Usability
- **Excellent (5):** Fluent navigation, independent parameter adjustment, feature discovery
- **Good (4):** Successful with minimal guidance, natural flow through tasks
- **Fair (3):** Some confusion but ultimately successful, needs moderate guidance
- **Poor (2):** Multiple assists necessary, struggles to find controls
- **Very Poor (1):** Cannot navigate tool effectively even with extensive help

### Engagement and Real-World Connection
- **Excellent (5):** Provides multiple examples, sees many use cases, asks insightful questions
- **Good (4):** Connects to at least one relevant scenario, explores beyond tasks
- **Fair (3):** Follows along with tasks, makes basic real-world connections
- **Poor (2):** Minimal engagement, doesn't see personal application
- **Very Poor (1):** Cannot connect to any real-world use, appears uninterested

## Special Adaptations

### For Non-Math Users
- Focus on visual patterns over equations
- Use more everyday analogies
- Emphasize practical applications
- Skip exports unless they ask

### For Advanced Users
- Discuss: "Euler integration" approximation quality
- Explore: Different numerical integration methods
- Challenge: "What if rate depends on current value?"
- Deepen: Financial/asset growth modeling

### For Teachers
- Ask about classroom demonstration value
- Discuss age-appropriate explanations
- Explore test question generation potential
- Consider curriculum tie-ins
