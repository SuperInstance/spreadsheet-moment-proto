# Part I: Spiking Neural Network Mapping

## 1.1 From Continuous to Spike-Based Inference

### Definition 1.1 (Rate Coding)

For a continuous activation $x \in \mathbb{R}$, the spike rate representation:

$$r = f(x) = \frac{r_{\max}}{1 + e^{-\beta x}}$$

where:
- $r_{\max}$ is the maximum firing rate (typically 100-1000 Hz)
- $\beta$ controls the steepness of the transfer function
- $x$ is the continuous activation value

### Definition 1.2 (Spike Train Generation)

A spike train $S(t)$ is a point process:

$$S(t) = \sum_{k} \delta(t - t_k)$$

where $t_k$ are spike times generated by a Poisson process with rate $r$.

### Theorem 1.1 (Ternary Weight Spike Propagation)

For a layer with ternary weights $W \in \{-1, 0, +1\}^{m \times n}$ and input spike trains $\{S_j(t)\}$:

The output spike rate for neuron $i$:

$$r_i^{out} = \sum_{j=1}^{n} W_{ij} \cdot r_j^{in} \cdot \epsilon_{ij}$$

where $\epsilon_{ij}$ is the synaptic efficacy factor.

**For Ternary Weights**:
- $W_{ij} = +1$: Excitatory synapse, $r_i^{out} += r_j^{in}$
- $W_{ij} = -1$: Inhibitory synapse, $r_i^{out} -= r_j^{in}$
- $W_{ij} = 0$: No connection, no contribution

### Proof Sketch

The post-synaptic potential (PSP) for neuron $i$:

$$V_i(t) = \sum_j W_{ij} \int_0^t \epsilon(t - t') S_j(t') dt'$$

For Poisson spike trains, the average PSP:

$$\langle V_i \rangle = \sum_j W_{ij} r_j^{in} \int_0^\infty \epsilon(\tau) d\tau = \sum_j W_{ij} r_j^{in} \tau_s$$

where $\tau_s$ is the synaptic time constant.

The output firing rate follows a transfer function:

$$r_i^{out} = \phi(V_i - V_{th})$$

For linear regime (far from saturation):

$$r_i^{out} \approx \alpha \sum_j W_{ij} r_j^{in}$$ $\square$

## 1.2 Integrate-and-Fire Neuron Model

### Definition 1.3 (Leaky Integrate-and-Fire)

The membrane potential $V(t)$ evolves as:

$$\tau_m \frac{dV}{dt} = -V(t) + R \cdot I(t)$$

with spike when $V(t) \geq V_{th}$, followed by reset $V \to V_{reset}$.

### Theorem 1.2 (Spike Rate for Ternary Input)

For constant input current from ternary weights:

$$I_i = \sum_{j: W_{ij} \neq 0} W_{ij} \cdot I_j^{in}$$

The steady-state firing rate:

$$r_i = \begin{cases}
0 & I_i \leq I_{th} \\
\left(\tau_{ref} + \tau_m \ln\frac{R I_i}{R I_i - V_{th}}\right)^{-1} & I_i > I_{th}
\end{cases}$$

where $I_{th} = V_{th}/R$.

### Corollary 1.1 (Ternary Weight Current)

For ternary synapses with spike rate coding:

$$I_{syn}^{ij} = W_{ij} \cdot q_{syn} \cdot r_j$$

where $q_{syn}$ is the synaptic charge per spike.

**Hardware Implementation**:
```
Current source per synapse:
  - W = +1: Current injection per spike
  - W = -1: Current extraction per spike  
  - W = 0: No connection

Total current: I_i = Σ_j W_{ij} × q_syn × r_j
```

## 1.3 Energy Savings from Sparse Spiking

### Definition 1.4 (Sparsity in Spiking)

For a spike train with rate $r$ and simulation time $T$:

$$\text{Sparsity} = 1 - \frac{\text{Number of spikes}}{\text{Maximum possible spikes}} = 1 - \frac{r \cdot T}{r_{\max} \cdot T} = 1 - \frac{r}{r_{\max}}$$

### Theorem 1.3 (Energy Scaling with Sparsity)

The energy for spike-based inference:

$$E_{spike} = E_{spike}^0 \cdot \langle s \rangle \cdot N_{syn}$$

where:
- $E_{spike}^0$ is the energy per spike event
- $\langle s \rangle$ is the average spike rate (normalized)
- $N_{syn}$ is the number of synapses

**Comparison to Continuous**:

$$\frac{E_{spike}}{E_{continuous}} = \frac{E_{spike}^0 \cdot \langle s \rangle}{E_{MAC}}$$

For typical values:
- $E_{spike}^0 \approx 1$ pJ (analog spike)
- $E_{MAC} \approx 10$ pJ (digital MAC)
- $\langle s \rangle \approx 0.1$ (10% average firing rate)

$$\frac{E_{spike}}{E_{continuous}} = \frac{1 \cdot 0.1}{10} = 0.01$$

**Result: 100× energy savings from sparse spiking.**

### Theorem 1.4 (Ternary Sparsity Advantage)

For ternary weights with inherent sparsity $s_0 = P(W_{ij} = 0)$:

The effective sparsity in spiking:

$$s_{eff} = s_0 + (1 - s_0) \cdot (1 - \langle r \rangle) = 1 - (1 - s_0)\langle r \rangle$$

For BitNet with $s_0 = 0.34$ and $\langle r \rangle = 0.1$:

$$s_{eff} = 1 - 0.66 \times 0.1 = 0.934$$

**93.4% of synapses are idle at any time.**

## 1.4 Spike Rate Equations for Transformer Layers

### Definition 1.5 (Spiking Transformer Layer)

For a transformer with spiking neurons:

**Linear Layer**:
$$r^{out} = \phi(W \cdot r^{in})$$

where $\phi$ is the spike rate transfer function.

**Attention Mechanism**:
$$r_i^{attn} = \sum_j A_{ij}(r) \cdot r_j^{value}$$

where attention weights $A_{ij}$ are computed from spike rates.

### Theorem 1.5 (Spike Rate Attention)

For spike rate coding, the attention weight:

$$A_{ij} = \frac{\exp(r_i^{query} \cdot r_j^{key} / \tau)}{\sum_k \exp(r_i^{query} \cdot r_k^{key} / \tau)}$$

**Approximation for Ternary Keys/Queries**:

For ternary $q_i, k_j \in \{-1, 0, +1\}^d$:

$$q_i \cdot k_j = \sum_{l=1}^d q_{il} k_{jl} = N_{++} - N_{+-} - N_{-+} + N_{--}$$

where $N_{ab}$ counts positions where $q_{il} = a$ and $k_{jl} = b$.

### Hardware Simplification

**Event-Driven Attention**: Only compute attention when spikes arrive:

$$A_{ij}(t) \to A_{ij}(t) \cdot \delta(t - t_j^{spike})$$

This reduces attention computation by factor $\langle r \rangle$.

# Part II: Memristive Weight Storage

## 2.1 MRAM as Ternary Memristor

### Definition 2.1 (Memristance)

A memristor has resistance that depends on the history of current:

$$M(q) = \frac{d\Phi}{dq}$$

where $\Phi$ is flux and $q$ is charge.

### Theorem 2.1 (MRAM Ternary States)

MRAM cells can encode ternary weights through resistance states:

$$R(W_{ij}) = \begin{cases}
R_{Low} & W_{ij} = +1 \\
R_{High} & W_{ij} = -1 \\
R_{Float} & W_{ij} = 0 \text{ (disconnected)}
\end{cases}$$

**Physical Implementation**:

| Weight | MRAM Configuration | Resistance | Current Direction |
|--------|-------------------|------------|-------------------|
| +1 | Parallel MTJ | Low (~1 kΩ) | Forward |
| -1 | Anti-parallel MTJ | High (~10 kΩ) | Reverse |
| 0 | Disabled cell | Open circuit | None |

### Definition 2.2 (MTJ - Magnetic Tunnel Junction)

The tunnel magnetoresistance:

$$TMR = \frac{R_{AP} - R_P}{R_P}$$

where $R_{AP}$ is anti-parallel resistance and $R_P$ is parallel resistance.

**Typical Values**: $TMR \approx 100-200\%$

### Theorem 2.2 (Memristance Equations for Ternary States)

For an MRAM cell in the ternary array:

**State +1 (Ferromagnetic Alignment)**:
$$M_{+1} = R_P = \frac{t_{ox}}{A \cdot G_P}$$

where $G_P$ is the conductance for parallel alignment.

**State -1 (Anti-parallel Alignment)**:
$$M_{-1} = R_{AP} = R_P (1 + TMR)$$

**State 0 (Disabled)**:
$$M_0 \to \infty \text{ (open circuit)}$$

## 2.3 Endurance and Retention Analysis

### Definition 2.4 (MRAM Endurance)

The number of write cycles before failure:

$$N_{cycles} = N_0 \cdot \exp\left(-\frac{E_a}{k_B T}\right)$$

where $E_a$ is the activation energy for breakdown.

**Typical MRAM Endurance**: $10^{12} - 10^{15}$ cycles

### Theorem 2.4 (Read/Write Endurance for Fixed Weights)

For mask-locked weights (read-only):

$$N_{read}^{max} \gg N_{write}^{max}$$

**For Fixed Inference Chip**:
- Write cycles: 1 (during manufacturing)
- Read cycles: Unlimited (no degradation)

**Conclusion**: Mask-locked MRAM has effectively infinite endurance for inference.

### Definition 2.5 (Data Retention)

Time for which data remains valid:

$$t_{retention} = \tau_0 \cdot \exp\left(\frac{\Delta E}{k_B T}\right)$$

where $\Delta E$ is the energy barrier for state switching.

**For MRAM**: $t_{retention} > 10$ years at 85°C

### Theorem 2.5 (Thermal Stability Factor)

The thermal stability factor:

$$\Delta = \frac{E_b}{k_B T} = \frac{M_s \cdot H_k \cdot V}{2 k_B T}$$

where:
- $M_s$ is saturation magnetization
- $H_k$ is anisotropy field
- $V$ is free layer volume

**Required**: $\Delta > 60$ for 10-year retention at 85°C

### Definition 2.6 (Sneak Path Current)

In a crossbar array, unwanted current paths through unselected cells:

$$I_{sneak} = \sum_{paths} \frac{V_{applied}}{\prod_{cells \in path} R_{cell}}$$

### Theorem 2.6 (Sneak Path Suppression for Ternary)

For ternary crossbar with $s_0 = P(W_{ij} = 0)$:

The sneak path probability:

$$P_{sneak} = (1 - s_0)^{L_{path}}$$

where $L_{path}$ is the sneak path length.

## 3.1 Event-Driven Attention Architecture

### Definition 3.1 (Spike-Triggered Attention)

Instead of computing attention for all token pairs, only compute when spikes arrive:

$$\text{Attention}(Q, K, V) \to \sum_{t \in \text{spike times}} \text{Attention}(Q(t), K, V)$$

### Theorem 3.1 (Spike Rate Attention Complexity)

For sequence length $L$ and average spike rate $\langle r \rangle$:

**Standard Attention**: $O(L^2 \cdot d)$ operations
**Spike-Triggered**: $O(L^2 \cdot d \cdot \langle r \rangle)$ operations

**Reduction Factor**: $1 / \langle r \rangle$

For $\langle r \rangle = 0.1$: 10× reduction in attention operations.

## 3.2 Timing Equations for Event-Driven Attention

### Definition 3.3 (Event Timing Model)

For a spike arriving at time $t_{spike}$:

**Event Processing Pipeline**:
1. **Detection**: $\tau_{det}$ (fast, ~ns)
2. **Key Fetch**: $\tau_{fetch}$ (memory access, ~10-100 ns)
3. **Similarity**: $\tau_{sim}$ (ternary MAC, ~1 ns per operation)
4. **Softmax**: $\tau_{softmax}$ (exponential, ~10 ns)
5. **Aggregation**: $\tau_{agg}$ (accumulate, ~1 ns)

**Total Latency**:
$$\tau_{event} = \tau_{det} + \tau_{fetch} + \tau_{sim} \cdot L + \tau_{softmax} + \tau_{agg}$$

### Theorem 3.2 (Latency vs. Accuracy Trade-off)

For spike-triggered attention with threshold $\theta$:

$$\text{Accuracy}(\theta) = A_0 \cdot (1 - e^{-\alpha/\theta})$$

where $\theta$ controls the spike threshold (higher $\theta$ = fewer spikes).

**Latency**:
$$\text{Latency}(\theta) = \tau_{base} + \beta \cdot N_{spikes}(\theta)$$

where $N_{spikes}(\theta) \sim 1/\theta$.

**Optimal Threshold**: Maximize accuracy under latency constraint.

$$\theta^* = \arg\max_\theta A_0 \cdot (1 - e^{-\alpha/\theta}) - \lambda \cdot (\tau_{base} + \beta/\theta)$$

### Definition 3.4 (Attention Spike Timing)

The timing of attention output spikes:

$$t_{out} = t_{spike} + \tau_{event} + \mathcal{N}(0, \sigma_t^2)$$

where $\sigma_t$ is timing jitter (typical ~1-10 ns).

**Synchronization**: Use clockless design for true event-driven operation.

## 3.3 Circuit Implementation of Attention

### Definition 3.5 (Ternary Dot Product Circuit)

For query vector $q$ and key vector $k$ with ternary elements:

```
    Q[i]     K[j]
      │       │
      ├──XOR──┤
      │   │   │
      │   ▼   │
      │  AND  │
      │   │   │

      │       │
      └───────┘
      
    q_i · k_j = Σ_l q_il ⊕ k_jl (for ternary)

**Ternary Dot Product**:
$$q \cdot k = |q \cap k| - |q \cap (-k)|$$

where $|q \cap k|$ is the number of matching signs.

### Theorem 3.3 (Efficient Ternary Similarity)

For ternary vectors $q, k \in \{-1, 0, +1\}^d$:

$$q \cdot k = \sum_{l=1}^d \text{sign}(q_l \cdot k_l) \cdot |q_l \cdot k_l|$$

**Simplification**:
- If $q_l = 0$ or $k_l = 0$: contribution = 0
- If $q_l = k_l \neq 0$: contribution = +1
- If $q_l = -k_l \neq 0$: contribution = -1

**Circuit**: XOR + AND + Population Count

# Part IV: Plasticity for Adaptation

## 4.1 On-Chip Learning in MRAM Layer

### Definition 4.1 (Plastic MRAM Layer)

5% of weights are stored in rewritable MRAM for on-chip adaptation:

$$W = W_{fixed} + W_{plastic}$$

where:
- $W_{fixed}$: Mask-locked ternary weights (95%)
- $W_{plastic}$: Rewritable MRAM weights (5%)

### Theorem 4.1 (Adaptive Capacity)

For a model with $N$ total parameters:

$$N_{plastic} = 0.05 \cdot N$$

**For 2B parameter model**: $N_{plastic} = 100$ M adaptable parameters.

**Learning Capacity**: Sufficient for domain adaptation, few-shot learning.

---

## 4.2 Spike-Timing Dependent Plasticity (STDP)

### Definition 4.2 (Classical STDP)

Weight update based on relative spike timing:

$$\Delta w = \begin{cases}
A_+ \cdot e^{-\Delta t / \tau_+} & \Delta t > 0 \text{ (pre before post)} \\
-A_- \cdot e^{\Delta t / \tau_-} & \Delta t < 0 \text{ (post before pre)}
\end{cases}$$

where $\Delta t = t_{post} - t_{pre}$.

### Theorem 4.2 (Ternary STDP Constraint)

For ternary weights, STDP must preserve discrete states:

$$W_{new} = \text{clip}(W_{old} + \Delta W, -1, +1) \in \{-1, 0, +1\}$$

**Update Rule**:
$$W_{new} = \begin{cases}
+1 & W_{old} = 0, \Delta W > \theta_+ \\
-1 & W_{old} = 0, \Delta W < -\theta_- \\
0 & W_{old} = \pm 1, |\Delta W| > \theta_{prune} \\
W_{old} & \text{otherwise}
\end{cases}$$

### Definition 4.3 (Probabilistic Ternary STDP)

Use probability of transition rather than threshold:

$$P(W_{new} = w' | W_{old} = w, \Delta t) = f_{w \to w'}(\Delta t)$$

**Transition Probabilities**:

| From | To +1 | To 0 | To -1 |
|------|-------|-------|-------|
| +1 | $1 - p_{prune}$ | $p_{prune}$ | 0 |
| 0 | $p_{+}(\Delta t)$ | $1 - p_{+} - p_{-}$ | $p_{-}(\Delta t)$ |
| -1 | 0 | $p_{prune}$ | $1 - p_{prune}$ |

where:
- $p_{+}(\Delta t) = A_+ e^{-|\Delta t|/\tau}$ for $\Delta t > 0$
- $p_{-}(\Delta t) = A_- e^{-|\Delta t|/\tau}$ for $\Delta t < 0$

## 4.3 Weight Update Rules for Ternary STDP

### Theorem 4.3 (Accumulated Evidence Model)

Maintain accumulator $A_{ij}$ for each plastic weight:

$$A_{ij} \leftarrow \alpha \cdot A_{ij} + (1 - \alpha) \cdot \Delta w_{ij}^{STDP}$$

**State Transition**:
$$W_{ij} = \begin{cases}
+1 & A_{ij} > A_{th} \\
-1 & A_{ij} < -A_{th} \\
0 & -A_{th} \leq A_{ij} \leq A_{th}
\end{cases}$$

### Definition 4.4 (MRAM Write Operation)

To update MRAM cell from state $W_{old}$ to $W_{new}$:

**Write Sequence**:
1. Read current state: $W_{old}$
2. If $W_{old} \neq W_{new}$:
   - Apply write pulse to switch MTJ state
   - Write energy: $E_{write} \approx 1-10$ pJ

### Theorem 4.4 (Energy Cost of Plasticity)

For learning rate $\eta$ and sequence length $L$:

$$E_{plasticity} = N_{plastic} \cdot P_{update} \cdot E_{write}$$

where $P_{update}$ is the probability of weight update per token.

**Estimation**:
- $N_{plastic} = 100$ M
- $P_{update} \approx 10^{-6}$ (sparse updates)
- $E_{write} \approx 5$ pJ

$$E_{plasticity} \approx 100 \times 10^6 \times 10^{-6} \times 5 \text{ pJ} = 500 \text{ pJ per token}$$

**Negligible compared to inference energy** (~60 μJ/token).

## 4.4 Local Learning Rules

### Definition 4.5 (Hebbian Learning for Ternary)

Classical Hebb's rule: "Neurons that fire together, wire together"

**Ternary Version**:
$$W_{ij}^{new} = \text{sign}(W_{ij}^{old} + \eta \cdot r_i \cdot r_j)$$

where $r_i, r_j$ are firing rates of pre- and post-synaptic neurons.

### Theorem 4.5 (Oja's Rule for Ternary)

Oja's rule normalizes weights automatically:

$$\Delta W_{ij} = \eta \cdot r_i \cdot r_j - \eta \cdot r_i^2 \cdot W_{ij}$$

**Ternary Constraint**: Apply sign function after update:

$$W_{ij}^{new} = \text{Ternarize}\left(W_{ij}^{old} + \Delta W_{ij}\right)$$

### Definition 4.6 (Three-Factor Learning Rule)

Incorporate a modulatory signal (reward/prediction error):

$$\Delta W_{ij} = \eta \cdot r_i \cdot r_j \cdot M$$

where $M$ is the modulatory factor (could be global or local).

**Application**: Reinforcement learning on-chip for task adaptation.

           └────────┬───────────────┘
                    │
                    ▼
           ┌─────────────────┐
           │  Timing Detector │
           │   Δt = t_post -  │
           │        t_pre     │
           └────────┬─────────┘
                    │
                    ▼
           ┌─────────────────┐
           │ STDP Function   │
           │ Δw = f(Δt)      │
           └────────┬─────────┘
                    │
                    ▼
           ┌─────────────────┐
           │ Accumulator     │
           │ A += Δw         │
           └────────┬─────────┘
                    │
                    ▼
           ┌─────────────────┐

           └─────────────────┘
```

### Theorem 4.6 (Circuit Timing Requirements)

For STDP with time constants $\tau_{pre}, \tau_{post} \sim 10-100$ ms:

**Trace Decay Circuit**: Requires analog or digital integrator with matching time constant.

**Timing Resolution**: Need $\Delta t$ resolution of ~1 ms for STDP curve.

**Implementation Options**:
1. Analog: RC circuit for trace decay
2. Digital: Counter-based timing with microsecond resolution

# Part V: Reservoir Computing with Fixed Weights

## 5.1 Mask-Locked Reservoir Architecture

### Definition 5.1 (Reservoir Computing)

A reservoir computing system consists of:
1. **Input Layer**: Projects input to reservoir
2. **Reservoir**: Fixed, random recurrent network
3. **Readout Layer**: Trainable linear output

$$\mathbf{r}(t) = (1-\alpha)\mathbf{r}(t-1) + \alpha \cdot \tanh(W_{in} \mathbf{u}(t) + W_{res} \mathbf{r}(t-1))$$

$$\mathbf{y}(t) = W_{out} \mathbf{r}(t)$$

### Theorem 5.1 (Mask-Locked Reservoir Configuration)

For our chip architecture:

| Component | Implementation | Trainability |
|-----------|---------------|--------------|
| Input Weights $W_{in}$ | Mask-locked ternary | Fixed |
| Reservoir Weights $W_{res}$ | Mask-locked ternary | Fixed |
| Readout Weights $W_{out}$ | Plastic MRAM | Trainable |

**Advantage**: Only 5% of weights need training (readout layer).

### Definition 5.2 (Echo State Property)

The reservoir has echo state property if it asymptotically forgets initial conditions:

$$\lim_{t \to \infty} \|\mathbf{r}_1(t) - \mathbf{r}_2(t)\| = 0$$

for any two initial states $\mathbf{r}_1(0), \mathbf{r}_2(0)$ driven by same input.

### Theorem 5.2 (Spectral Radius Condition)

The echo state property holds if:

$$\rho(W_{res}) < 1$$

where $\rho(\cdot)$ is the spectral radius.

**For Ternary Reservoir**:
With ternary weights, the spectral radius can be tuned by scaling:

$$W_{res}^{scaled} = \gamma \cdot W_{res}^{ternary}$$

where $\gamma < 1/\rho(W_{res}^{ternary})$.

---

## 5.2 Lyapunov Stability Analysis

### Definition 5.3 (Lyapunov Exponent)

The largest Lyapunov exponent measures sensitivity to initial conditions:

$$\lambda_{max} = \lim_{t \to \infty} \frac{1}{t} \ln \frac{\|\delta \mathbf{r}(t)\|}{\|\delta \mathbf{r}(0)\|}$$

**Stability Criterion**: $\lambda_{max} < 0$ for stable reservoir dynamics.

### Theorem 5.3 (Lyapunov Exponent for Ternary Reservoir)

For reservoir dynamics with ternary weights:

$$\lambda_{max} \approx \ln(\rho(W_{res})) - \langle \text{saturation} \rangle$$

where saturation comes from the tanh nonlinearity.

**Design Rule**: Choose reservoir scale such that:

$$\lambda_{max} \in (-0.5, 0)$$

for rich dynamics with stability.

### Definition 5.4 (Memory Capacity)

The memory capacity measures how much past input the reservoir can store:

$$MC = \sum_{k=1}^{\infty} MC_k$$

where $MC_k$ is the correlation between output and input $k$ steps ago.

### Theorem 5.4 (Memory Capacity Bound)

For reservoir with $N$ neurons:

$$MC \leq N$$

**For Ternary Reservoir**:
Effective memory capacity depends on the spectral radius and sparsity.

$$MC_{ternary} \approx N \cdot (1 - s_0) \cdot (1 - \rho)$$

where $s_0 = P(W_{ij} = 0)$ is the sparsity.

**For $N = 2048$, $s_0 = 0.34$, $\rho = 0.9$**:
$$MC \approx 2048 \times 0.66 \times 0.1 \approx 135$$

---

## 5.3 Fixed-Weight Reservoir Dynamics

### Definition 5.5 (Reservoir States for Transformer)

Use the mask-locked layers as a deep reservoir:

$$\mathbf{r}_l = f_l(\mathbf{r}_{l-1}; W_l^{fixed})$$

where each layer is a reservoir stage.

### Theorem 5.5 (Hierarchical Reservoir Computing)

Multi-layer reservoir provides hierarchical feature extraction:

$$\text{Features at layer } l: \quad \mathbf{h}_l = [\mathbf{r}_1, \mathbf{r}_2, \ldots, \mathbf{r}_l]$$

**Readout from any layer**: Train separate readout weights for each layer.

### Definition 5.6 (Readout Training)

Train only the plastic MRAM weights in the readout layer:

$$W_{out}^* = \arg\min_{W_{out}} \sum_t \|y(t) - W_{out} r(t)\|^2 + \lambda \|W_{out}\|^2$$

**Closed-form solution (Ridge Regression)**:
$$W_{out} = Y R^T (R R^T + \lambda I)^{-1}$$

where $R$ is reservoir state matrix and $Y$ is target output matrix.

## 5.4 Performance Analysis

### Theorem 5.6 (Readout Learning Efficiency)

For on-chip learning with plastic readout:

**Memory**: $O(N_{readout}) = O(0.05 \cdot N_{total})$

**Computation**: $O(N_{readout} \cdot T)$ for $T$ training samples

**Convergence**: Ridge regression has closed-form solution (instant for small readout).

### Definition 5.7 (Task Suitability)

Reservoir computing is most suitable for:

2. **Time series prediction**: Financial, weather, control
3. **Anomaly detection**: Continuous monitoring

**Less suitable for**: Static pattern recognition (use full fine-tuning instead)

# Part VI: Neural Synchrony and Oscillations

## 6.1 Emergent Oscillations in PE Array

### Definition 6.1 (Oscillatory Dynamics)

Each Processing Element (PE) in the systolic array can exhibit oscillatory behavior:

$$\frac{d\theta_i}{dt} = \omega_i + \sum_{j=1}^{N} K_{ij} \sin(\theta_j - \theta_i)$$

where $\theta_i$ is the phase of PE $i$ and $\omega_i$ is its natural frequency.

### Theorem 6.1 (Kuramoto Model for PE Array)

For $N = 1024$ PEs in a 32×32 systolic array, the collective dynamics follow the Kuramoto model:

$$\frac{d\theta_i}{dt} = \omega_i + \frac{K}{N} \sum_{j=1}^{N} \sin(\theta_j - \theta_i)$$

where $K$ is the coupling strength.

**Order Parameter**:
$$r e^{i\psi} = \frac{1}{N} \sum_{j=1}^{N} e^{i\theta_j}$$

### Definition 6.2 (Synchronization Transition)

The critical coupling for synchronization:

$$K_c = \frac{2}{\pi g(0)}$$

where $g(\omega)$ is the distribution of natural frequencies.

**For Uniform Distribution** on $[-\Delta, \Delta]$:
$$K_c = \frac{4\Delta}{\pi}$$

### Theorem 6.2 (PE Synchronization Regimes)

| Coupling $K$ | Behavior | Inference Impact |
|-------------|----------|------------------|
| $K < K_c$ | Incoherent | Independent PE timing |
| $K \approx K_c$ | Critical | Optimal for adaptive computation |
| $K > K_c$ | Synchronized | Coordinated array operation |

---

## 6.2 Synchronization for Coordinated Computation

### Definition 6.3 (Phase-Locked Computation)

PEs coordinate computation through phase locking:

$$\theta_i(t) = \Omega t + \phi_i$$

where $\Omega$ is the collective frequency and $\phi_i$ is the relative phase.

### Theorem 6.3 (Phase-Based Scheduling)

Use phase differences for computational scheduling:

$$\text{PE}_i \text{ active when } \theta_i \in [\theta_{start}, \theta_{end}]$$

**Advantages**:
- Distributed timing (no global clock)
- Adaptive computation speed
- Energy proportional to activity

### Definition 6.4 (Oscillatory Attention)

Use oscillations to gate attention computation:

$$A_{ij}(t) = A_{ij}^0 \cdot \cos^2\left(\frac{\theta_i(t) - \theta_j(t)}{2}\right)$$

**Interpretation**: PEs in phase have stronger attention coupling.

### Theorem 6.4 (Frequency Coding)

Different frequencies encode different feature types:

| Frequency Band | Feature Type | Analogous to |
|---------------|--------------|--------------|
| Low (1-10 Hz) | Slow features | Delta waves (sleep) |
| Medium (10-50 Hz) | Standard features | Alpha/Beta (awake) |
| High (50-200 Hz) | Fine details | Gamma (attention) |

---

## 6.3 Kuramoto Model for Chip Dynamics

### Definition 6.5 (Extended Kuramoto Model)

For PEs with local coupling topology (systolic array neighbors):

$$\frac{d\theta_i}{dt} = \omega_i + K \sum_{j \in \mathcal{N}_i} \sin(\theta_j - \theta_i) + \xi_i(t)$$

where $\mathcal{N}_i$ is the neighborhood of PE $i$ and $\xi_i(t)$ is noise.

### Theorem 6.5 (Convergence to Synchronization)

For identical oscillators ($\omega_i = \omega_0$) with all-to-all coupling:

$$\lim_{t \to \infty} r(t) = 1 \quad \text{(Full synchronization)}$$

For local coupling (systolic array):

$$\lim_{t \to \infty} r(t) = r_{\infty} < 1$$

where $r_{\infty}$ depends on coupling strength and topology.

### Definition 6.6 (Chimera States)

Coexistence of synchronized and desynchronized regions:

$$r_i^{local} = \left|\frac{1}{|\mathcal{N}_i|} \sum_{j \in \mathcal{N}_i} e^{i\theta_j}\right|$$

**Chimera**: Some regions have high $r_i^{local}$, others low.

**Application**: Different PE regions can operate at different speeds.

---

## 6.4 Hardware Implementation of Oscillations

### Definition 6.7 (Ring Oscillator PE)

Each PE contains a ring oscillator for local timing:

```
    ┌─────────────────────────────────┐
    │                                 │
    │   ┌───┐   ┌───┐   ┌───┐        │
    │   │NOT├──►│NOT├──►│NOT├──┐     │
    │   └───┘   └───┘   └───┘  │     │
    │                          │     │
    └──────────────────────────┘     │
                 ▲                   │
                 └───────────────────┘
                 
    Ring oscillator provides:
    - Local clock for PE
    - Phase θ_i for synchronization
    - Natural frequency ω_i
```

### Theorem 6.6 (Phase Coupling Circuit)

Implement Kuramoto coupling through current injection:

$$I_{coupling,i} = K \sum_j \sin(\theta_j - \theta_i)$$

**Circuit**:
```
    PE_j phase ──►┌───────┐
                  │ Phase │
                  │Diff.  │──► Current ──┐
                  │(sin)  │    Source    │
    PE_i phase ──►└───────┘              │
                                          ▼
                                  Ring Oscillator
                                  (frequency shift)
```

**Result**: Phase difference → Current → Frequency adjustment → Synchronization

### Definition 6.8 (Global vs. Local Synchronization)

| Mode | Coupling | Benefits | Drawbacks |
|------|----------|----------|-----------|
| **Global Sync** | All-to-all | Uniform timing | High wiring cost |
| **Local Sync** | Nearest-neighbor | Efficient wiring | Phase gradients |
| **Hierarchical** | Cluster + global | Best of both | Complex design |

**Recommendation**: Local coupling for systolic array (natural for 32×32 grid).

import numpy as np
from scipy.integrate import odeint
import matplotlib.pyplot as plt

class NeuromorphicSimulator:
    """
    Simulate neuromorphic behavior for ternary inference chip
    """
    
    def __init__(self, n_pe=1024, weight_sparsity=0.34):
        self.n_pe = n_pe
        self.weight_sparsity = weight_sparsity
        self.weights = self._generate_ternary_weights()
        
    def _generate_ternary_weights(self):
        """Generate BitNet-style ternary weight matrix"""
        probs = [self.weight_sparsity, 
                 (1-self.weight_sparsity)/2, 
                 (1-self.weight_sparsity)/2]
        return np.random.choice([-1, 0, 1], 
                               size=(int(np.sqrt(self.n_pe)), 
                                     int(np.sqrt(self.n_pe))),
                               p=[probs[2], probs[0], probs[1]])
    
    def spike_rate_transfer(self, x, r_max=1000, beta=1.0):
        """Rate coding transfer function"""
        return r_max / (1 + np.exp(-beta * x))
    
    def lif_neuron(self, V, I, tau_m=10e-3, V_th=1.0, V_reset=0.0):
        """Leaky integrate-and-fire dynamics"""
        dV = (-V + I) / tau_m
        spike = V >= V_th
        V_new = np.where(spike, V_reset, V + dV)
        return V_new, spike
    
    def stddp_update(self, dt, A_plus=0.1, A_minus=0.1, 
                     tau_plus=20e-3, tau_minus=20e-3):
        """STDP weight update for timing difference dt"""
        if dt > 0:
            return A_plus * np.exp(-dt / tau_plus)
        else:
            return -A_minus * np.exp(dt / tau_minus)
    
    def ternary_stdp(self, w_old, dw, theta=0.5):
        """Ternary-constrained STDP"""
        w_new = w_old + dw
        if w_new > theta and w_old == 0:
            return 1
        elif w_new < -theta and w_old == 0:
            return -1
        elif abs(w_new) < 0.5 and w_old != 0:
            return 0
        return w_old
    
    def kuramoto_dynamics(self, theta, t, omega, K, N):
        """Kuramoto model for PE synchronization"""
        dtheta = np.zeros_like(theta)
        for i in range(N):
            dtheta[i] = omega[i] + (K/N) * np.sum(np.sin(theta - theta[i]))
        return dtheta
    
    def simulate_synchronization(self, T=10.0, dt=0.01, K=2.0):
        """Simulate PE array synchronization"""
        N = self.n_pe
        omega = np.random.normal(1.0, 0.1, N)  # Natural frequencies
        theta0 = np.random.uniform(0, 2*np.pi, N)
        
        t = np.arange(0, T, dt)
        theta = odeint(self.kuramoto_dynamics, theta0, t, 
                       args=(omega, K, N))
        
        # Order parameter
        r = np.abs(np.mean(np.exp(1j * theta), axis=1))
        
        return t, theta, r
    
    def reservoir_dynamics(self, u, W_res, W_in, r0=None, alpha=0.5):
        """Echo state network dynamics"""
        T = len(u)
        N = W_res.shape[0]
        r = np.zeros((T, N))
        r[0] = r0 if r0 is not None else np.zeros(N)
        
        for t in range(1, T):
            r[t] = (1-alpha)*r[t-1] + alpha*np.tanh(W_in @ u[t] + W_res @ r[t-1])
        
        return r
    
    def compute_energy(self, n_spikes, e_spike=1e-12, n_mac=1e9, e_mac=10e-12):
        """Compare energy: spiking vs digital"""
        E_spike = n_spikes * e_spike
        E_digital = n_mac * e_mac
        return E_spike, E_digital, E_digital / E_spike


# Run simulation
sim = NeuromorphicSimulator()

# 1. Synchronization
t, theta, r = sim.simulate_synchronization(T=5.0, K=2.5)
print(f"Final order parameter: r = {r[-1]:.3f}")

# 2. Energy comparison
spike_rate = 0.1  # 10% average firing rate
n_neurons = 1024
n_spikes = int(n_neurons * spike_rate * 100)  # 100 timesteps
E_spike, E_digital, savings = sim.compute_energy(n_spikes)
print(f"Energy savings: {savings:.1f}x")

# 3. Reservoir dynamics
W_res = np.random.randn(512, 512) * 0.9 / np.sqrt(512)
W_in = np.random.randn(512, 64)
u = np.random.randn(100, 64)
r = sim.reservoir_dynamics(u, W_res, W_in)
print(f"Reservoir state shape: {r.shape}")
