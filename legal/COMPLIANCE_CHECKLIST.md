## 3. Data Subject Request Handling

### 3.1 Access Requests
- [ ] Verify identity of requestor
- [ ] Locate all personal data
- [ ] Compile personal data
- [ ] Provide copy within 30 days (GDPR) or 45 days (CCPA)
- [ ] Provide information about processing activities
- [ ] Document request and response

### 3.2 Deletion Requests
- [ ] Verify identity of requestor
- [ ] Determine if deletion is required (or exception applies)
- [ ] Delete personal data from all systems
- [ ] Delete personal data from backups (where feasible)
- [ ] Notify third parties where data was shared (if required)
- [ ] Confirm deletion to requestor
- [ ] Document request and response

### 3.3 Rectification Requests
- [ ] Verify identity of requestor
- [ ] Verify inaccuracy of personal data
- [ ] Correct inaccurate personal data
- [ ] Complete incomplete personal data
- [ ] Notify third parties where data was shared (if required)
- [ ] Confirm correction to requestor
- [ ] Document request and response

### 3.4 Portability Requests
- [ ] Verify identity of requestor
- [ ] Compile personal data
- [ ] Provide data in structured, commonly used, machine-readable format
- [ ] Provide data within 30 days
- [ ] Document request and response

### 3.5 Restriction Requests
- [ ] Verify identity of requestor
- [ ] Determine if restriction is required
- [ ] Mark data as restricted in systems
- [ ] Prevent further processing (except storage)
- [ ] Notify requestor
- [ ] Document request and response

### 3.6 Objection Requests
- [ ] Verify identity of requestor
- [ ] Determine if objection is valid
- [ ] Stop processing (if objection is valid)
- [ ] Notify requestor
- [ ] Document request and response

## 4. Data Breach Management

### 4.1 Detection and Identification
- [ ] Detect potential breach
- [ ] Identify affected systems
- [ ] Identify affected data
- [ ] Identify affected individuals
- [ ] Assess severity and risk

### 4.2 Containment
- [ ] Immediately contain breach
- [ ] Prevent further unauthorized access
- [ ] Preserve evidence
- [ ] Document breach details

### 4.3 Assessment
- [ ] Investigate cause of breach
- [ ] Determine scope of breach
- [ ] Assess risk to individuals
- [ ] Determine if notification is required

### 4.4 Notification

#### 4.4.1 Supervisory Authority (EU/UK)
- [ ] Notify within 72 hours of becoming aware
- [ ] Provide required information:
  - [ ] Nature of breach
  - [ ] Categories of data and individuals affected
  - [ ] Likely consequences
  - [ ] Measures taken to address breach
  - [ ] Measures taken to mitigate adverse effects

#### 4.4.2 Data Subjects (EU/UK)
- [ ] Notify without undue delay if high risk
- [ ] Provide required information:
  - [ ] Nature of breach
  - [ ] Likely consequences
  - [ ] Measures taken
  - [ ] Recommendations for mitigating adverse effects

#### 4.4.3 Other Jurisdictions
- [ ] Comply with local breach notification requirements
- [ ] California: As soon as possible
- [ ] Canada: If real risk of significant harm
- [ ] Australia: Under Notifiable Data Breaches scheme
- [ ] Other: As required by local law

### 4.5 Remediation
- [ ] Implement measures to prevent recurrence
- [ ] Update security procedures
- [ ] Conduct post-incident review
- [ ] Document lessons learned
- [ ] Update breach response procedures

## Jurisdiction-Specific Checklists

### GDPR (European Union)

#### Article 30 Records
- [ ] Name and contact details of controller and DPO
- [ ] Purposes of processing
- [ ] Categories of data subjects and personal data
- [ ] Categories of recipients
- [ ] International transfers and safeguards
- [ ] Retention periods
- [ ] Security measures

#### DPIAs (Article 35)
- [ ] Conduct DPIA for high-risk processing
- [ ] Systematic and extensive evaluation
- [ ] Large-scale processing of special category data
- [ ] Systematic monitoring
- [ ] Consult supervisory authority if high risk remains

#### Data Subject Rights (Articles 15-22)
- [ ] Right to be informed (Articles 13-14)
- [ ] Right of access (Article 15)
- [ ] Right to rectification (Article 16)
- [ ] Right to erasure (Article 17)
- [ ] Right to restriction of processing (Article 18)
- [ ] Right to data portability (Article 20)
- [ ] Right to object (Article 21)
- [ ] Rights related to automated decision making (Article 22)

## Emergency Procedures

### Data Breach Emergency
1. **Immediate Response (0-1 hour)**
   - [ ] Contain breach
   - [ ] Activate incident response team
   - [ ] Preserve evidence
   - [ ] Begin investigation

2. **Assessment (1-24 hours)**
   - [ ] Determine scope of breach
   - [ ] Identify affected data and individuals
   - [ ] Assess risk level
   - [ ] Determine notification requirements

3. **Notification (24-72 hours)**
   - [ ] Notify supervisory authority (if required)
   - [ ] Notify affected individuals (if required)
   - [ ] Notify other stakeholders (if required)

4. **Remediation (ongoing)**
   - [ ] Implement security improvements
   - [ ] Document lessons learned
   - [ ] Update procedures

### Security Emergency
1. **Immediate Response (0-1 hour)**
   - [ ] Activate security team
   - [ ] Isolate affected systems
   - [ ] Preserve evidence
   - [ ] Begin investigation

2. **Assessment (1-24 hours)**
   - [ ] Determine scope of incident
   - [ ] Identify affected systems and data
   - [ ] Assess impact
   - [ ] Determine if breach occurred

3. **Response (ongoing)**
   - [ ] Remediate security issue
   - [ ] Restore services
   - [ ] Monitor for recurrence
   - [ ] Document incident
