**Topic:** Building 10,000-User Real-Time Collaboration

**Title:** Scaling Real-Time Collaboration to 10,000+ Users: Lessons from Building Spreadsheet Moment

Spreadsheet Moment broke through this ceiling by implementing custom CRDTs (Conflict-Free Replicated Data Types) optimized for spreadsheet semantics, enabling 10,000+ simultaneous users with sub-50ms synchronization latency.

This session will dive deep into the technical architecture that makes massive collaboration possible:

**Technical Deep Dive:**
- **CRDT Implementation:** Custom CRDTs designed for spreadsheet data models
- **Performance Optimization:** Rust-based microservices for <50ms operations
- **Edge Computing:** 300+ locations for global synchronization
- **Conflict Resolution:** Mathematical guarantees of convergence
- **Scalability:** From 1 to 10,000 users without performance degradation

**Key Takeaways:**
- Why operational transformation doesn't scale past ~100 users
- How CRDTs enable massive real-time collaboration
- Performance optimization techniques for billion-cell spreadsheets
- Edge computing architecture for global latency
- Lessons from scaling to 10,000 concurrent users

**Target Audience:**
- Software engineers building real-time collaboration features
- Architects designing distributed systems
- Developers interested in CRDTs and conflict-free replication
- Technical leaders scaling collaboration infrastructure

**Prerequisites:**
- Familiarity with real-time collaboration concepts
- Understanding of distributed systems basics
- Interest in performance optimization

**Topic:** Academic Paper on Scaling Collaboration

**Title:** Scaling Real-Time Collaboration to 10,000+ Users: CRDTs, Edge Computing, and the Future of Cooperative Work

**Abstract (Academic Paper):**
Real-time collaboration platforms traditionally use Operational Transformation (OT) for synchronization, which introduces algorithmic complexity that limits scalability to ~100 concurrent users. This paper presents Spreadsheet Moment's approach to scaling collaboration to 10,000+ simultaneous users through custom CRDTs optimized for spreadsheet semantics.

**Research Contributions:**

1. **CRDT Design for Spreadsheets:**
   - Custom CRDT data types for spreadsheet cells, rows, columns
   - Optimized convergence algorithms for formula dependencies
   - Efficient encoding for billion-cell workbooks
   - Performance evaluation: <50ms operations at 10,000-user scale

2. **Edge Computing Architecture:**
   - Hierarchical edge network with 300+ locations globally
   - Adaptive routing based on user geography and load
   - Sub-50ms synchronization latency p95 worldwide
   - Cost optimization through intelligent edge placement

3. **User Experience at Scale:**
   - User study with 1,000+ concurrent participants
   - Qualitative feedback on massive collaboration experience
   - Comparison with traditional ~100-user platforms
   - Implications for future collaborative systems

4. **Performance Characterization:**
   - Benchmark suite for collaborative editing workloads
   - Scalability analysis from 1 to 10,000 concurrent users
   - Network efficiency and bandwidth utilization
   - Computational overhead of CRDT metadata

**Key Findings:**
- CRDTs enable 100x scale improvement over OT
- Edge computing essential for global latency targets
- User experience quality maintained at 10,000-user scale
- Open challenges in persistence and replay at scale

**Target Audience:**
- Academic researchers in CSCW
